# Granny — Google Sheets Database Schema

## Overview

Six sheets (tabs) form a normalized relational model:

| Sheet | Role |
|---|---|
| `Users` | Multi-user auth & permissions |
| `People` | One row per individual |
| `Families` | One row per couple/union |
| `Family_Children` | Junction: child ↔ family (with relationship type) |
| `Events` | One row per discrete event |
| `Event_Participants` | Junction: person ↔ event (with role) |

---

## ID Conventions

All primary keys are **string prefixes + zero-padded integers**, generated by Apps Script.

| Sheet | PK format | Example |
|---|---|---|
| Users | `USR####` | `USR0001` |
| People | `PER####` | `PER0042` |
| Families | `FAM####` | `FAM0007` |
| Events | `EVT####` | `EVT0019` |

Junction tables use a composite key (not stored) — the combination of their two FK columns is the logical PK.

Apps Script ID generator:
```js
function nextId(sheet, prefix) {
  const lastRow = sheet.getLastRow();
  if (lastRow < 2) return prefix + '0001';
  const lastId = sheet.getRange(lastRow, 1).getValue();
  const num = parseInt(lastId.replace(prefix, '')) + 1;
  return prefix + String(num).padStart(4, '0');
}
```

## Sheet 2: `People`

One row per individual. Birth and death are stored here as **attributes**, not as Events, for fast lookup. The Events sheet handles the richer, sourced versions.

| Column | Type | Notes |
|---|---|---|
| `person_id` | String PK | `PER####` |
| `first_name` | String | |
| `middle_name` | String | |
| `last_name` | String | Birth surname |
| `sex` | String | `M` / `F` / `U` (unknown) |
| `birth_date` | String | ISO 8601 or partial: `1923`, `1923-04`, `1923-04-15` |
| `birth_place` | String | Free text |
| `death_date` | String | Same partial-date convention |
| `death_place` | String | |
| `is_living` | Boolean | `TRUE`/`FALSE` — controls privacy |
| `notes` | String | Free text |
| `created_by` | String FK | → `Users.user_id` |
| `created_at` | DateTime | |
| `updated_at` | DateTime | |

## Sheet 3: `Families`

One row per **union** (couple). Represents the partnership itself, not the children. A person appears here as `spouse1` or `spouse2`; they appear in `Family_Children` as a child.

| Column | Type | Notes |
|---|---|---|
| `family_id` | String PK | `FAM####` |
| `spouse1_id` | String FK | → `People.person_id` (nullable) |
| `spouse2_id` | String FK | → `People.person_id` (nullable; omit for single-parent families) |
| `union_type` | String | `married` / `partnered` / `unknown` |
| `union_date` | String | Partial-date ok |
| `union_place` | String | |
| `union_end_date` | String | |
| `union_end_reason` | String | `divorce` / `death` / `annulment` |
| `notes` | String | |
| `created_by` | String FK | → `Users.user_id` |
| `created_at` | DateTime | |
| `updated_at` | DateTime | |

**Key design notes:**
- A person with **two marriages** has two rows in `Families` (e.g., `FAM0001` and `FAM0005`), appearing as spouse in both.
- A **blended family** is modeled by multiple `Families` rows sharing some of the same children (via `Family_Children`).
- Single-parent: leave `spouse2_id` blank.

---

## Sheet 4: `Family_Children`

Junction table linking a child (`People`) to the family unit (`Families`) they belong to as a child. One person can appear multiple times here (biological + adoptive families).

| Column | Type | Notes |
|---|---|---|
| `family_id` | String FK | → `Families.family_id` |
| `child_id` | String FK | → `People.person_id` |
| `relationship_type` | String | `biological` / `adopted` / `step` / `foster` |
| `notes` | String | |

**Composite PK:** `(family_id, child_id, relationship_type)` — same child can link to same family twice only if relationship_type differs.

**Example — blended family:**
```
FAM0001: spouse1=PER0001 (Alice), spouse2=PER0002 (Bob)   ← Alice's 1st marriage
FAM0002: spouse1=PER0003 (Carol), spouse2=PER0002 (Bob)   ← Bob's 2nd marriage (blended)

Family_Children:
FAM0001 | PER0010 | biological   ← Alice & Bob's child
FAM0002 | PER0011 | biological   ← Carol & Bob's child
FAM0002 | PER0010 | step         ← Alice's child now in Bob+Carol's home
```

---

## Sheet 6: `Event_Participants`

Junction table linking people to events with a role.

| Column | Type | Notes |
|---|---|---|
| `event_id` | String FK | → `Events.event_id` |
| `person_id` | String FK | → `People.person_id` |
| `role` | String | `subject` / `spouse` / `parent` / `child` / `witness` / `informant` |
| `notes` | String | |

**Composite PK:** `(event_id, person_id)`

**Example — 1920 Census:**
```
Events:
EVT0042 | census | 1920 | Cook County IL | "1920 US Census — Smith household"

Event_Participants:
EVT0042 | PER0001 | subject    ← head of household
EVT0042 | PER0002 | spouse
EVT0042 | PER0010 | child
EVT0042 | PER0011 | child
```

## FK Relationship Map

```
Users ──────────────────────────────────┐
  user_id                               │ created_by (all sheets)
                                        ▼
People ──────────┬──────────────────────┤
  person_id      │                      │
                 │ spouse1_id           │
                 │ spouse2_id           │
                 ▼                      │
Families         │                      │
  family_id ─────┼──── Family_Children  │
                 │       family_id ─────┘
                 │       child_id → People.person_id
                 │
People           │
  person_id ─────┴──── Event_Participants
                         person_id → People.person_id
                         event_id  → Events.event_id
```

## Assembling Nested JSON in Apps Script

When the Vue frontend requests a person profile, Apps Script reads all sheets once into memory, then assembles the nested object.

```js
function getPersonProfile(personId) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();

  // Load all sheets into arrays-of-objects
  const people  = sheetToObjects(ss.getSheetByName('People'));
  const families = sheetToObjects(ss.getSheetByName('Families'));
  const famChildren = sheetToObjects(ss.getSheetByName('Family_Children'));
  const events  = sheetToObjects(ss.getSheetByName('Events'));
  const evtParts = sheetToObjects(ss.getSheetByName('Event_Participants'));

  const person = people.find(p => p.person_id === personId);
  if (!person) return null;

  // --- Parent families (families where this person is a child) ---
  const parentLinks = famChildren.filter(fc => fc.child_id === personId);
  person.parent_families = parentLinks.map(link => {
    const fam = families.find(f => f.family_id === link.family_id);
    return {
      ...fam,
      relationship_type: link.relationship_type,
      spouse1: fam.spouse1_id ? people.find(p => p.person_id === fam.spouse1_id) : null,
      spouse2: fam.spouse2_id ? people.find(p => p.person_id === fam.spouse2_id) : null,
    };
  });

  // --- Spouse families (families where this person is a spouse) ---
  const spouseFamilies = families.filter(
    f => f.spouse1_id === personId || f.spouse2_id === personId
  );
  person.spouse_families = spouseFamilies.map(fam => {
    const partnerId = fam.spouse1_id === personId ? fam.spouse2_id : fam.spouse1_id;
    const children = famChildren
      .filter(fc => fc.family_id === fam.family_id)
      .map(fc => ({ ...people.find(p => p.person_id === fc.child_id), relationship_type: fc.relationship_type }));
    return {
      ...fam,
      partner: partnerId ? people.find(p => p.person_id === partnerId) : null,
      children,
    };
  });

  // --- Events ---
  const participations = evtParts.filter(ep => ep.person_id === personId);
  person.events = participations.map(ep => ({
    ...events.find(e => e.event_id === ep.event_id),
    role: ep.role,
    other_participants: evtParts
      .filter(x => x.event_id === ep.event_id && x.person_id !== personId)
      .map(x => ({ ...people.find(p => p.person_id === x.person_id), role: x.role })),
  }));

  return person;
}

// Converts a sheet to an array of {header: value} objects
function sheetToObjects(sheet) {
  const [headers, ...rows] = sheet.getDataRange().getValues();
  return rows
    .filter(r => r[0] !== '')
    .map(r => Object.fromEntries(headers.map((h, i) => [h, r[i]])));
}
```

## Multi-User Access Control

The `Users` sheet + `created_by` columns give you an audit trail. For access control in Apps Script:

- Use `Session.getActiveUser().getEmail()` to identify the caller.
- Look up their `role` in the `Users` sheet.
- Return `403` (as a JSON error) for viewers attempting writes.

```js
function getCurrentUser() {
  const email = Session.getActiveUser().getEmail();
  const users = sheetToObjects(SpreadsheetApp.getActiveSpreadsheet().getSheetByName('Users'));
  return users.find(u => u.email === email) || null;
}
```
